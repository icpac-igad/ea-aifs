# CEDA ERA5T Input for AIFSgaia Ensemble Forecasting

## Overview

This document describes how to create AIFS input states from CEDA ERA5T data,
matching the requirements of the AIFSgaia model as described in the AI Weather
Quest submission.

## AIFSgaia Input Requirements

From the AIFSgaia model description:
https://aiweatherquest.ecmwf.int/team/aifs/
> For real-time forecasting, AIFSgaia uses two consecutive states (t, t-24h)
> from ERA5T and its associated EDA as input.

**Input variables:**

| Category | Variables | Source |
|----------|-----------|--------|
| Pressure levels (13) | temperature, humidity, geopotential height, horizontal winds | ERA5T operational |
| Surface | 10m winds, 2m temperature, mean sea level pressure, total column water vapour | ERA5T EDA (10 members) |
| Forcing | orography, land-sea mask, insolation | Static / computed |

**Pressure levels:** 1000, 925, 850, 700, 600, 500, 400, 300, 250, 200, 150, 100, 50 hPa

**Temporal structure:** Two timesteps per field — `t` and `t-24h` — stacked as a `(2, N_gridpoints)` array on the N320 reduced Gaussian grid (542,080 points).

**Ensemble strategy:** 10 EDA members provide perturbed surface fields. Ensemble diversity in forecasts is further generated by conditioning on random noise during inference.

## CEDA Data Mapping

### Surface fields — per-member from `enda/an_sfc`

CEDA path: `https://data.ceda.ac.uk/badc/ecmwf-era5t/data/enda/an_sfc/YYYY/MM/DD/`

File naming: `ecmwf-era5t_enda_an_sfc_YYYYMMDDHHMM.memN.PARAM.nc`

| AIFSgaia variable | CEDA parameter | CEDA variable name | Status |
|-------------------|----------------|-------------------|--------|
| 10m U-wind | `10u` | `u10` | Available, per-member |
| 10m V-wind | `10v` | `v10` | Available, per-member |
| 2m temperature | `2t` | `t2m` | Available, per-member |
| Mean sea level pressure | `msl` | `msl` | Available, per-member |
| Total column water vapour | `tcwv` | `tcwv` | Available, per-member |

10 ensemble members: `mem0` through `mem9`.

### Pressure level fields — deterministic from `oper/an_ml`

CEDA path: `https://data.ceda.ac.uk/badc/ecmwf-era5t/data/oper/an_ml/YYYY/MM/DD/`

File naming: `ecmwf-era5t_oper_an_ml_YYYYMMDDHHMM.PARAM.nc`

CEDA does not archive pressure-level data directly. Instead, model-level (L137)
data is available and must be interpolated to standard pressure levels.

| Variable | CEDA param | Model levels | Notes |
|----------|-----------|-------------|-------|
| Temperature | `t` | 137 levels | ~213 MB per file |
| Specific humidity | `q` | 137 levels | ~98 MB per file |
| U-wind | `u` | 137 levels | ~213 MB per file |
| V-wind | `v` | 137 levels | ~213 MB per file |
| Log surface pressure | `lnsp` | surface | ~1.6 MB, needed for interpolation |
| Surface geopotential | `z` | surface | ~1.3 MB, needed for geopotential computation |

**Geopotential at pressure levels** is not stored directly. It must be computed
from `t`, `q`, surface `z`, and `lnsp` using the hypsometric equation (see below).

### Data characteristics

- Grid: 0.25° regular lat-lon (721 x 1440), already on 0-360° longitude
- Regridded to N320 reduced Gaussian grid (542,080 points) using `earthkit-regrid`
- Each field becomes a `(2, 542080)` array after stacking t-24h and t

## Model Level to Pressure Level Interpolation

### ECMWF L137 Hybrid Coordinates

Pressure at each model level is computed from:

```
p_half(k) = a(k) + b(k) * sp    for k = 0..137 (138 half-levels)
p_full(k) = (p_half(k-1) + p_half(k)) / 2    for k = 1..137
```

where `sp = exp(lnsp)` is surface pressure and `a(k)`, `b(k)` are the L137
hybrid coefficients from ECMWF (source: [L137 model level definitions](https://confluence.ecmwf.int/display/UDOC/L137+model+level+definitions)).

### Geopotential via Hypsometric Equation

Since `z` on model levels is not archived (only surface geopotential), we
compute geopotential at each model level by integrating upward from the surface:

```
Tv(k) = T(k) * (1 + (1/eps - 1) * q(k))     # virtual temperature
Phi(137) = Phi_sfc + Rd * Tv(137) * ln(p_half(137) / p_full(137))
Phi(k) = Phi(k+1) + Rd * Tv(k+1) * ln(p_full(k+1) / p_half(k+1))
                   + Rd * Tv(k) * ln(p_half(k+1) / p_full(k))
```

where `Rd = 287.058 J/(kg·K)` and `eps = 0.621971`.

### Interpolation to Pressure Levels

Linear interpolation in log-pressure space between the two bracketing model
levels for each target pressure level at each grid point.

## Ensemble Member Strategy

The deterministic pressure level fields (from `oper/an_ml`) are **shared
identically** across all 10 ensemble members. Each member's pkl file contains:

- **5 surface fields** (unique per member from EDA)
- **65 pressure level fields** (identical across members from operational analysis)
- **Total: 70 fields per member**

This reflects the AIFSgaia design where ensemble diversity primarily comes from:
1. Perturbed initial surface conditions (from 10 EDA members)
2. Stochastic noise injection during autoregressive inference

## Script Usage

```bash
# Run with uv (PEP 723 — auto-installs dependencies)
uv run ceda_era5t_pkl_input_aifsens.py
```

Requires a `.env` file with the CEDA access token:

```
ceda_token=<your-token>
```

Token obtained from: https://services-beta.ceda.ac.uk/account/token/

Authentication uses wget with Bearer token header per
[CEDA download guide](https://help.ceda.ac.uk/article/5191-downloading-multiple-files-with-wget).

## Output Format

Each member produces a pickle file matching the format expected by the AIFS
inference pipeline:

```python
{
    "date": datetime(2026, 2, 27, 0, 0),
    "fields": {
        "10u": np.array(shape=(2, 542080)),   # EDA member-specific
        "10v": np.array(shape=(2, 542080)),
        "2t":  np.array(shape=(2, 542080)),
        "msl": np.array(shape=(2, 542080)),
        "tcwv": np.array(shape=(2, 542080)),
        "t_1000": np.array(shape=(2, 542080)),  # shared deterministic
        "t_925":  np.array(shape=(2, 542080)),
        ...
        "z_50":  np.array(shape=(2, 542080)),
        "v_50":  np.array(shape=(2, 542080)),
    }
}
```

## Comparison with ECMWF Open Data Approach

| Aspect | ECMWF Open Data script | CEDA ERA5T script |
|--------|----------------------|-------------------|
| Data source | ECMWF Open Data (ENS) | CEDA ERA5T archive (EDA + oper) |
| Ensemble members | 50 (ENS) | 10 (EDA) — matches AIFSgaia design |
| Surface fields | `ecmwf-open-data` Python package | wget + Bearer token from CEDA |
| Pressure levels | Directly available on pressure levels | Interpolated from 137 model levels |
| Temporal offset | t and t-6h | t and t-24h — matches AIFSgaia spec |
| tcwv vs tcw | tcw (total column water) | tcwv (total column water vapour) — matches AIFSgaia spec |
| Longitude convention | -180 to 180 (needs shift) | 0 to 360 (no shift needed) |
| Data fidelity | Operational ENS (lower res, different DA) | ERA5T reanalysis + EDA (matches training data) |

The CEDA approach is more faithful to AIFSgaia's training configuration, which
used ERA5 reanalysis with 10 EDA members.
